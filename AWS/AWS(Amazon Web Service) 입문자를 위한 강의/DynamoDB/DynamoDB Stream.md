# DynamoDB Stream
- DynamoDB 테이블에서 일어나는 일들(삽입, 수정, 삭제 등)이 일어날 시 시간적 순서에 맞게 Streams에 기록
- Log는 즉각 암호화가 일어나며 24시간동안 보관됨
- 주로 이벤트를 기록하고 이벤트 발생을 외부로 알리는 용도 (예시 : Lambda Function)
- 이벤트 전&후에 대한 상황 보관
- 예를 들어, 온라인 쇼핑몰을 운영한다고 하자. 한 고객이 쇼핑몰에서 아이템을 구매할 때 송장 번호를 만들어준다고 하자. 송장 번호가 만들어지면 DynamoDB에 저장된다. 
  이러한 데이터 삽입 역시 해당 이벤트를 `DynamoDB Stream` 이 인식하고 Lambda 함수가 발동한다. 새로운 송장 번호가 생길 때마다 Lambda 함수가 실행될 것이다. 
  물건 구매가 이루어진 것을 감지한 Lambda 함수는 SNS 로 메시지를 전송시킨다. SNS 는 메시지를 생성하여 SQS 로 보낸다. 그러면 물건을 구매한 소비자는 결재창을 보게 될 것이고 
  결재창에 뜨는 정보는 SNS 로부터 가져온 메시지를 보게 될 것이다. 본인의 카드 정보를 기입하고 결재 버튼을 누르면 비로소 구매가 완료될 것이다. 이처럼, `DynamoDB Stream`은 
  DynamoDB에 이벤트가 발생되었을 때 대응할 수 있는 효율적인 아키텍처를 만드는 것을 가능하게 해준다.
